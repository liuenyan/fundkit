# 已知问题

## 分级基金净值折算未处理

### 问题描述

曾为分级基金的基金（如 165312 建信央视财经50指数）发生过**份额折算**（定期折算 / 不定期折算 / 分级转 LOF），折算日 `单位净值` 会大幅跳变（如 2.02 → 1.0），但投资者持有的份额会相应调整——总资产不变。

当前 `simulate_dca()` 直接用 `unit_nav` 计算市值（`units × nav`），折算日 `unit_nav` 下降但 `units` 未同步调整，导致组合市值**错误跳变**。

### 影响范围

- 回测区间若**包含折算日期**（2017-11-22 不定期折算、历年定期折算），结果不准确
- 回测区间在折算日**之后**（如 2024 年起），数据本身已经过调整，不受影响

### 检测方法

折算日特征：`unit_nav` 变幅 > 15% 且 `acc_nav` 变幅 < 2%（累计净值反映真实收益，折算日几乎不变）。

折算比例 = `prev_unit_nav / current_unit_nav`。

### 示例：165312

| 日期 | unit_nav | acc_nav | 变动 | 事件 |
|------|----------|---------|------|------|
| 2017-11-21 | 2.0240 | 2.1433 | — | 折算前 |
| 2017-11-22 | 1.0000 | 2.1433 | -50.6% | 不定期折算 |

### 修复思路

在 `simulate_dca()` 的每日循环中检测折算事件，若命中则：
1. `units ×= prev_unit_nav / current_unit_nav`
2. 等比例调整 `cost`、`fee_batches` 中的份额基数
3. 总资产不变，后续交易日恢复正常计算

### 涉及基金

所有曾为分级基金的 LOF 都可能存在此问题，如：
- 165312 建信央视财经50指数

## 分红与净值折算 API 调研结果

### 基金分红 API

| API | 用途 | 165312 结果 |
|-----|------|-------------|
| `fund_open_fund_info_em(symbol, indicator="分红送配详情")` | 分红事件（每份分红金额） | 返回空 DataFrame |
| `fund_fh_em()` | 分红排行（全市场） | 不包含 165312 |
| `fund_announcement_dividend_em(symbol)` | 分红配送公告标题列表 | 可查到折算公告标题，**无折算比例** |

### 分级基金 API

| API | 用途 | 结果 |
|-----|------|------|
| `fund_graded_fund_info_em(symbol)` | 分级基金净值历史明细 | 返回与普通基金相同字段（`unit_nav` / `acc_nav`），**无折算事件标识** |
| `fund_graded_fund_daily_em` | 分级基金日线（全市场） | 未测试（需遍历全市场） |

### 结论

**不存在直接返回折算比例/参数的 AKShare API。** 公告 API 只有标题和报告 ID，要获取折算比例需要单独抓取公告详情页（非结构化 HTML），成本高且不稳定。

因此修复应走 **净值数据自身检测** 路线：
- 利用 `unit_nav` 与 `acc_nav` 在折算日的差异行为自动检测
- 折算比例 = `prev_unit_nav / current_unit_nav`
- 零外部 API 依赖，适用于所有曾为分级基金的标的
