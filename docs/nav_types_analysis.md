# 净值口径对均线策略的影响分析

> 三种净值（单位净值/累计净值/复权净值）在定投均线策略中的行为差异
> 测试基金: 100032（中证红利，高分红）

## 三种净值的定义

| 名称 | 公式 | 分红日行为 | 含义 |
|------|------|-----------|------|
| **单位净值** | 基金资产/总份额 | **跳降**（降幅=分红额） | 当前交易单价 |
| **累计净值** | 单位净值 + 每份累计分红 | **不变**（降价+加回抵消） | 持有至今未再投资的总回报 |
| **复权净值** | 从日增长率链式累积 | **随日增长率变化** | 分红再投资的总回报 |

## 分红日的实际数据（100032）

以 2026-02-10 分红日为例：

```
            单位净值    累计净值    复权净值    日增长率
2026-02-09   1.0680    3.3150    5.0070    +0.75%
2026-02-10   1.0010    3.3180    5.0210    +0.28%  ← 除息
变化：       -0.0670   +0.0030   +0.0140
```

- **单位净值** 跳降 -0.067（约 -6.3%），纯属分红导致
- **累计净值** 无跳降（+0.0030），分红额加回到累计，与降价抵消
- **复权净值** 无跳降（+0.0140），日增长率已是复权口径

## 对均线策略偏离度的影响

均线策略的核心判断依据：

```
偏离度 = (当前净值 - 250日均线) / 250日均线
偏离度 < -10% → 严重低估 → 2.0倍买入
偏离度 > +5%  → 高估     → 停投
```

### 使用单位净值的问题

高分红基金经过多次分红，单位净值被系统性压低。以 100032 为例：

- 成立 17 年，累计分红约 2.306 元/份
- 每次除息 unit_nav 跳降一次，MA 被逐步拖低
- **当前（2026-06-29）**：

```
口径       当前值   MA250     偏离度      策略判断
─────────────────────────────────────────────────
单位净值   0.9090  1.0150   -10.45%  → 严重低估 → 2.0x 加倍买入 ❌
累计净值   3.2150  3.2832    -2.08%  → 正常范围 → 1.0x 基准定投 ✅
复权净值   4.5608  4.8742    -6.43%  → 偏低     → 1.5x 加码买入
```

**-10.45% 是假信号**——不是市场真的低估，而是 17 年分红积累的下跌被均线滞后了。按此信号加倍买入，会导致在已经不算便宜的市场位置进一步加重仓位，增加回撤风险。

### 使用累计净值的优势

- **无分红跳降**：除息日累计净值不降，MA 不被污染
- **偏离度真实**：-2.08% 反映的是真实市场位置，不是分红干扰
- **对低分红基金无影响**：创业板/沪深300/中证500 的 unit_nav 与 acc_nav 偏差分布差距 < 0.1%

### 使用复权净值的情况

复权净值考虑了分红再投资的复利效应，增长速度更快。100032 的数据：

| 百分位 | 累计净值 | 复权净值 |
|--------|---------|---------|
| 10%（极端低估） | -5.3% | **-8.1%** |
| 50%（中位数） | +1.5% | **+3.5%** |
| 90%（极端高估） | +7.0% | **+12.1%** |

复权口径的偏差分布比累计口径约宽 1.5 倍。若改用复权净值，5 档阈值需同步外扩（如 -10% → -15%）以保持策略行为一致。

## 当前实现选择

本工具统一使用**累计净值**作为均线计算口径：

1. 消除分红跳降导致的假信号
2. 偏离度阈值保持默认（-10%/-5%/0/+5%）无需调整
3. 低分红基金表现完全不变
4. 累计净值直接来自数据源，无需额外计算

### 回退机制

当输入数据不含 `acc_nav` 列时（如单元测试、旧数据），自动回退到 `unit_nav`：

```python
col = "acc_nav" if "acc_nav" in nav_df.columns else "unit_nav"
```

## 未来扩展：基于跟踪指数计算均线

上述方案解决的是**基金本身**净值的平滑问题。对于指数基金，还有更精确的思路——直接用基金跟踪的**指数价格**来计算均线。

### 理论依据

```
基金 unit_nav  =  指数成分股总市值 + 现金  ─  基金自身分红
                                                  ↑
                                             基金自身分红只影响基金，
                                             不影响指数
```

指数价格不含基金自身分红，且成分股除权在时间上分散，不会导致单日跳降。因此：

- **指数基金场景**：用跟踪指数算 MA 最精确，完全消除分红干扰
- **主动基金场景**：仍用基金累计净值算 MA（没有跟踪标的）

两层都有意义：
| 场景 | MA 基于 | 偏离度意义 |
|------|---------|-----------|
| 指数基金 + 跟踪指数价格 | 指数 MA | **市场估值** — 指数相对自身历史均值的偏离 |
| 主动基金 / 无指数数据 | 基金累计净值 MA | **基金自身趋势** — 基金表现相对自身历史均值的偏离 |

### 数据源

AKShare `stock_zh_index_daily_em` 可获取指数历史日线（含前收盘价）：

```python
import akshare as ak

df = ak.stock_zh_index_daily_em(symbol="sh000922")  # 中证红利
```

需要基金→指数的映射关系，可从 `fund_profile.跟踪标的` 表获取。

### 架构接口

```python
class MovingAverageBuyStrategy(BuyStrategy):
    def __init__(self, base_amount, period, purchase_rate,
                 nav_df, index_df=None, ...):
        if index_df is not None:
            # 基于指数价格算 MA
            series = index_df.set_index("date")["close"]
        else:
            # 回退到基金累计净值
            series = nav_df.set_index("date")["acc_nav"]
        self.ma_series = series.rolling(period).mean()
```
